# API and CLI reference

This reference documents the stable API patterns used by the dashboard, CLI, and
automation clients. It intentionally omits secrets and environment-specific
values.

## Authentication

Most API routes call `getRequestAuth()` in `src/lib/request-auth.ts`.

Authentication is resolved in this order:

1. `Authorization: Bearer tg_pat_...` API token.
2. Encrypted dashboard session cookie.

Bearer tokens are generated by `POST /api/tokens`, stored only as SHA-256 hashes,
and returned in plaintext once. Tokens may expire, may be revoked, and update
`lastUsedAt` on successful use.

Example:

```bash
curl -H "Authorization: Bearer tg_pat_your_token" \
  https://tiny.garden/api/sites
```

## Token management

Token creation currently requires a dashboard session.

| Route | Auth | Behavior |
| --- | --- | --- |
| `GET /api/tokens` | Session | List token metadata for the signed-in user. Plaintext tokens are never returned. |
| `POST /api/tokens` | Session | Create a token. Body accepts `name` and optional `expiresInDays`; max expiry is 3650 days. |
| `DELETE /api/tokens/[id]` | Session or token | Revoke a token owned by the authenticated user. |

Example create request from a browser session:

```bash
curl -X POST https://tiny.garden/api/tokens \
  -H "Content-Type: application/json" \
  -d '{"name":"Local CLI","expiresInDays":90}'
```

## Site routes

| Route | Methods | Notes |
| --- | --- | --- |
| `/api/sites` | `GET`, `POST` | List owned sites or create a site from `{ channelSlug, channelTitle?, template, subdomain }`. Template must exist on disk. |
| `/api/sites/[id]` | `PATCH`, `DELETE` | Update `published`, `template`, or `channelSlug`; delete a site and remove its custom domain if present. |
| `/api/sites/[id]/build` | `POST` | Rebuild immediately. Enforces cooldown and daily quota unless the user is admin/friend. |
| `/api/sites/[id]/export` | `GET` | Return site URL and build metadata. |
| `/api/sites/[id]/preview/...` | `GET` | Owner-only preview of the last successful build. Does not require the site to be published. |
| `/api/sites/[id]/theme` | `GET`, `PUT` | Read/save theme colors and font tokens. |
| `/api/sites/[id]/css` | `GET`, `PUT` | Read/save custom site CSS and inspect channel `styles.css`. |
| `/api/sites/[id]/domain` | `GET`, `POST`, `DELETE` | Manage custom domain state and Vercel DNS verification. |

Create a site:

```bash
curl -X POST https://tiny.garden/api/sites \
  -H "Authorization: Bearer tg_pat_your_token" \
  -H "Content-Type: application/json" \
  -d '{
    "channelSlug": "my-archive",
    "channelTitle": "My Archive",
    "template": "gallery",
    "subdomain": "my-archive"
  }'
```

Trigger a rebuild:

```bash
curl -X POST https://tiny.garden/api/sites/site_id/build \
  -H "Authorization: Bearer tg_pat_your_token"
```

## Channel and template routes

| Route | Methods | Notes |
| --- | --- | --- |
| `/api/channels?source=own` | `GET` | User-owned Are.na channels. |
| `/api/channels?source=groups` | `GET` | Groups/teams and their channels. |
| `/api/channels?source=following` | `GET` | Channels the user follows or collaborates on. |
| `/api/channels/search?q=term` | `GET` | Are.na channel search; requires at least 2 characters. |
| `/api/templates` | `GET` | Templates from `templates/*/meta.json`, sorted by display name. |
| `/api/templates/preview` | `GET`, `POST` | Render preview HTML from mock data, configured example channel, or a user site. |

## Public site search

`GET /api/sites/search` supports public discovery and signed-in search.

Query parameters:

- `scope`: `mine`, `public`, or `all`. Defaults to `mine`.
- `q`: optional text query.
- `limit`: 1-50, default 20.
- `page`: 1-based page number.

`scope=public` only returns published sites and does not require auth.

```bash
curl "https://tiny.garden/api/sites/search?scope=public&q=gallery&limit=10"
```

## Build limits

Manual rebuild requests are recorded in `BuildRequest` and enforced by
`src/lib/build-limits.ts`.

| Plan | Accepted manual builds per UTC day |
| --- | ---: |
| Free | 10 |
| Pro | 100 |
| Studio | 500 |

There is also a 5-minute cooldown per site. Admin and friend users bypass these
manual build limits.

## CLI

The CLI uses the same Bearer-token API path as other automation clients. See
`cli/README.md` for command details.

Common flow:

```bash
tg auth login --token tg_pat_your_token
tg new --channel my-archive --template gallery --subdomain my-archive
tg sites
tg refresh my-archive
```

## Constraints and pitfalls

- Do not log or store plaintext `tg_pat_...` tokens after creation.
- `POST /api/tokens` is session-only because the token secret is shown once to
  the signed-in user.
- A site template must be a slug discovered from disk; unknown slugs are
  rejected on create, update, and build.
- Rebuild failures set `lastBuildError` on the site and return a 500 for manual
  requests.
- `POST /api/sites` starts the initial build in `after()` and returns the site
  record immediately; the static output may finish shortly after the API
  response.
